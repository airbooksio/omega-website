# Airbooks Website — Wireframes

Static HTML wireframes for the Airbooks site. **Structure and copy only — final design, fonts, colors, and graphics are not represented here.**

## Files

- `index.html` — Home (`/`)
- `platform.html` — Platform / OMEGA (`/platform`)
- `products.html` — Products (`/products`)
- `company.html` — Company (`/company`)
- `investors.html` — Investors with gated form (`/investors`)
- `contact.html` — Contact / demo (`/contact`)
- `styles.css` — Shared wireframe stylesheet (neutral grays only)

## How to view

Open `index.html` in any browser. No build step required.

## Choices baked in (so you have something concrete to react to)

| Decision | Choice in this wireframe |
|---|---|
| Hero subhead | Variant A — "Modern workflow applications for fixed-income capital markets, built on open infrastructure you control." |
| Brand spelling | Airbooks |
| Product names | Real names: Postgame, Thunderwriter Pro, Buyside Suite, Sync Bridge |
| Partner mentions | "Active deployment with top-tier broker-dealers and a leading electronic trading platform" — no names |
| Sitemap | Six pages (Home, Platform, Products, Company, Investors, Contact) |
| Investor page | Dedicated page + gated "Request investor materials" form |

## Visual placeholders

Anywhere you see a dashed-border gray box (e.g. `[ Hero visual: high-level OMEGA architectural diagram ]`), that's where final art / screenshots / diagrams go. Source diagrams from the pitch deck where noted.

## Annotations

Italic gray boxes labelled "Note:" call out things that need supplied content (bios, headshots, specific numbers, etc.) or are conditional (e.g. press section is optional).

## Not included

- Final typography (using system fonts in the wireframe)
- Brand color palette
- Animations, interactions, hover states
- Real form handlers (forms `alert()` on submit)
- Any graphics beyond placeholder boxes
